Variante 1 — Default · 3 tamaños (sm / md / lg)

Barra estándar con ícono lupa izquierda, botón limpiar y botón de búsqueda. El anillo de foco cumple visibilidad de estado (Nielsen #1). El botón X aparece solo al escribir para no contaminar la UI vacía.

Variante 2 — Con filtro de categoría

Selector de categoría anclado izquierda. Mejora eficiencia de búsqueda (Nielsen #7: flexibilidad). Aplica en e-commerce, portales documentales y buscadores avanzados.

Variante 3 — Hero Search (fondo marca)

Search como acción principal de la página. Ideal en portales de conocimiento, help centers e intranets. Incluye términos populares como reconocimiento rápido (Nielsen #6).

React TypeScript Props

type SearchBarSize = 'sm' | 'md' | 'lg';

interface SearchSuggestion {
  type: 'history' | 'result';
  title: string | React.ReactNode; // supports <mark> highlights
  meta?: string;
  badge?: string;
  href?: string;
}

interface SearchBarProps {
  size?: SearchBarSize;
  /** Renders pill (radius-full) instead of rounded-md */
  pill?: boolean;
  placeholder?: string;
  value?: string;
  disabled?: boolean;
  /** Category filter options shown on the left */
  filterOptions?: Array<{ value: string; label: string }>;
  /** Live autocomplete suggestions */
  suggestions?: SearchSuggestion[];
  onSearch?: (query: string) => void;
  onChange?: (value: string) => void;
  onClear?: () => void;
  className?: string;
}

/** Full-bleed hero variant wrapping SearchBar */
interface SearchHeroProps {
  eyebrow?: string;
  title?: string;
  searchBarProps?: SearchBarProps;
  popularTerms?: Array<{ label: string; href: string }>;
}
